Because of rest-assured 4.3.0 upgrade the Groovy from 2.5.7 to 3.0.2, see rest-assured/changelog.txt at master rest-assured/rest-assured, you can specified the groovy version in pom file.

\n 
 <properties>\n <rest-assured.version>4.3.0</rest-assured.version>\n <groovy.version>3.0.2</groovy.version>\n
    </properties>\n

I also had an java.lang.AbstractMethodError but on another method.

\n\n

rest-assured 4.3 upgraded it's implementation to use groovy 3, see rest assured's release notes

\n\n

In my case the dependency resolution downgraded groovy to use some 2.x version.

\n\n

If you use maven:

\n\n

You can check that using mvn dependency:tree. \nIf there's a conflict, add the necessary groovy dependencies as direct dependencies with a version=>3.0.2 to your project.

\n\n

If you use gradle:

\n\n

You can check that using gradle depencencies.\nIf there's a conflict, then add a resolution strategy:

\n\n 
configurations.all {\n    resolutionStrategy.eachDependency { DependencyResolveDetails details ->\n        if (details.requested.group == 'org.codehaus.groovy') {\n            details.useVersion \"3.0.2\"\n            details.because \"needed by rest-assured>=4.3\"\n        }\n    }\n}\n
\n\n

Maybe that can help somebody.

    <dependency>\n <groupId>io.rest-assured</groupId>\n <artifactId>spring-mock-mvc</artifactId>\n
    <scope>test</scope>\n <exclusions>\n <exclusion>\n <artifactId>groovy</artifactId>\n
    <groupId>org.codehaus.groovy</groupId>\n </exclusion>\n </exclusions>\n </dependency>\n
    <dependency>\n <groupId>org.codehaus.groovy</groupId>\n <artifactId>groovy</artifactId>\n
